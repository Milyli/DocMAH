** First Time Help and Creation **

* Prerequisites
1) jQuery
2) jQueryUi
3) jsTree (for documentation page only)
3) SqlServer compatible database



* Implementation
1) Configure moodule and handler in web.config.
<configuration>
	<system.web>
		<httpModules>
			<add name="DocMAH" type="DocMAH.Web.HttpModule, DocMAH"/>
		</httpModules>
		<httpHandlers>
			<add path="DocMAH.axd" verb="GET,POST" type="DocMAH.Web.HttpHandler, DocMAH"/>
		</httpHandlers>
	</system.web>
	<system.webServer>
		<modules>
				<add name="DocMAH" type="DocMAH.Web.HttpModule, DocMAH" preCondition="integratedMode" />
		</modules>
		<handlers>
				<add name="DocMAH" path="DocMAH.axd" verb="GET,POST" type="DocMAH.Web.HttpHandler, DocMAH" preCondition="integratedMode"/>
		</handlers>
	<system.webServer>
</configuration>

1.5) Two ways to set connection string. 1) {preferred} web.config (see next)
				- or -
		{programmatic ->} Configurator.ConnectionString = [connection string]
		used when connection string is only available programmatically. (super enterprisey API accessability type thing)


2) Configure docmah custom configuration values.
<configuration>
	<configSections>
		<section name="docmah" type="DocMAH.Configuration.DocmahConfigurationSection, DocMAH" />
	</configSections>
	
	<!-- docmah configuration section - Top docmah configuration element. -->
	<!-- connectionStringName [Optional] Name of connection string for database connections. If not set, must be set programmatically via DocMAH.Configuration.Configurator.ConnectionString on app startup.-->
	<!-- jsUrl [Optional - Default: renders individual links to MS CDN and cdnjs javascript libraries] Can override to a single URL (bundle) that contains jQuery, jQueryUi and jsTree script files. -->
	<!-- cssUrl [Optional - Default: renders individual link to cdnjs CDN css library] Can override to a single URL (bundle) that contains jsTree css default files. -->
	<docmah connectionStringName="[Connection String Name]" 
			  jsUrl=""
			  cssUrl="" > 
		
		<!-- documentation element - [Optional] Settings particular to the documentation page. -->
		<!-- pageTitle [Optional - Default: Documentation] Provide a custom title for documentation page. -->
		<documentation pageTitle="Documentation" 
		/>
		
		<!-- editHelp element - [Optional] Controls the availability of edit mode for first time help and documentation. -->
		<!-- requireAuthentication [Optional - Default: true] Require authentication to edit help content. -->
		<!-- requireLocalConnection [Optional - Default: true] Require a local connection to the server to edit help content. -->
		<!-- disabled [Optional - Default: false] Disable all help editing for the application. Recommended for production environments. -->
		<editHelp requireAuthentication="true" 
				  requireLocalConnection="true" 
				  disabled="false"
		/>
	
	</docmah>
</configuration>


3) [Optional, but recommended] Add a help button (input, div, image, anchor, any element that supports jQuery click) to your pages or layout with the class .dmh-view-button.

This either shows the help for the page or opens the documentation if a page is not available.


4) Match Urls support multiple space separated urls with wildcard (*).

For instance, a match urls value that works for all possible paths to the default ASP.NET MVC Home.Index action would be:
		/ /Home /Home/ /Home/Index*

All URLs must be unique in the database across all pages.
